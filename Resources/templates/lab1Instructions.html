Lab class 1: modifying JWT algorithm type to `none`

Authorization in this window is vulnerable to CVE-2020-15957. You can easily escalate user's privileges. If you know any user's credentials, your can become "admin" without knowing the admin's password.

Just sign in as "jim" with password "12345". Watch the cookies. You will find that one of the cookies stores JWT data. If you decode the claim part, you will see that it carries the login of currently authenticated user. What happens if you manipulate value of field "user"? And what happens, if you change the JWT header to use signature's algorithm to "none"? Don't forget to remove the signature part from JWT after modification. For JWT manipulation use base64url. Rember that JWT is base64Url, not base64! Be pentester! Authenticate yourself as admin without knowing the password. For manipulating cookie value use either browser's "Developer tools" or any browser's extension e.g. EditThisCookie

Presented on this page authorization process is for demostration purposees only, but remember that there are real apps with this vulnerability in the wildness of internet. Make sure your app is secure! Learn how to test against CVE-2020-15957.
